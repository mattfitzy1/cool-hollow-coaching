# System: Profit Discovery Audit

> Takes an uploaded P&L plus a customer/service breakdown sheet and surfaces all five areas of Mark's Profit Discovery Audit framework: pricing gaps, cost inefficiencies, customer profitability, service mix, and revenue leakage. In-program tool for Milestone 4 of Business Without You.

Distinct from `apps/hidden-profit-analyzer`, the prospect-facing lead magnet (see `docs/hidden-profit-analyzer.md`). This tool is for clients already in the program and also takes a customer/service breakdown sheet, so it can run two checks (customer profitability, service mix) the lead magnet cannot.

## Architecture

```
[CSV/Excel P&L upload] --> [analysis.py: load_pnl + 5 checks]
[CSV/Excel breakdown upload] --> [analysis.py: load_breakdown]    --> [Streamlit report in browser]
```

## Key Files

| File | Purpose |
|------|---------|
| `apps/profit-discovery-audit/app.py` | Streamlit web app: two file uploads, renders the five-check report, serves the branded template via a download button |
| `apps/profit-discovery-audit/analysis.py` | Core logic: normalizes the P&L and breakdown sheet, runs five checks |
| `apps/profit-discovery-audit/Cool_Hollow_Coaching_Milestone_4_Profit_Discovery_Audit_Template.xlsx` | Branded client-facing input template (P&L + Customer Service Breakdown tabs), generated by `apps/_shared/generate_templates.py` |
| `apps/profit-discovery-audit/sample_pnl.csv` | Internal dev-testing fixture, not client-facing |
| `apps/profit-discovery-audit/sample_breakdown.csv` | Internal dev-testing fixture, not client-facing |
| `apps/profit-discovery-audit/requirements.txt` | Pinned dependency versions |

## How It Works

1. User uploads a CSV or Excel P&L (line item name plus one amount column or one column per month) and a customer/service breakdown sheet (`type`, `name`, `revenue`, `direct_cost`).
2. `load_pnl()` normalizes the P&L into `category` (revenue, cogs, expense, leakage, other), `line_item`, `month`, `amount`, guessing category from keywords in the line item name.
3. `load_breakdown()` normalizes the breakdown sheet, validating the four required columns.
4. Five checks run independently, each returning a dollar estimate and plain-English findings: pricing gaps (gross margin vs. a 50% benchmark), cost inefficiencies (expense lines growing faster than 15% month over month, sorted chronologically), customer profitability and service mix (line margin vs. a 20% benchmark), and revenue leakage (discount/write-off/refund lines).
5. The app sums all five estimates into one headline dollar figure.

## Configuration

No API keys needed. Runs entirely locally on Python + Streamlit + pandas + openpyxl.

## Common Operations

**Start the app:**
```bash
cd apps/profit-discovery-audit
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

**Test the analysis logic directly:**
```bash
source .venv/bin/activate
python3 -c "
import pandas as pd
from analysis import run_full_audit
pnl = pd.read_csv('sample_pnl.csv')
breakdown = pd.read_csv('sample_breakdown.csv')
print(run_full_audit(pnl, breakdown))
"
```
