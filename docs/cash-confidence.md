# System: Cash Confidence

> Takes an uploaded 13-week cash item list and a recurring expense list, produces a rolling 13-week cash forecast, scores every expense against the five-question Financial Decision Filter, and shows the forecast impact of cutting flagged expenses. An optional third sheet adds a receivables-timing check: how much cash is stuck past the client's own payment terms. In-program tool for Milestone 5 of Business Without You.

## Architecture

```
[CSV/Excel cash items + recurring expenses + optional receivables upload] --> [analysis.py: forecast + decision filter + receivables check] --> [Streamlit report with two charts]
```

## Key Files

| File | Purpose |
|------|---------|
| `apps/cash-confidence/app.py` | Streamlit web app: three file uploads (one optional), charts, filter results, serves the branded template via a download button |
| `apps/cash-confidence/analysis.py` | Core logic: builds the weekly forecast, scores expenses, builds the adjusted forecast, scores receivables aging |
| `apps/cash-confidence/Cool_Hollow_Coaching_Milestone_5_Cash_Confidence_Template.xlsx` | Branded client-facing input template (Cash Items + Recurring Expenses + Receivables Aging tabs), generated by `apps/_shared/generate_templates.py` |
| `apps/cash-confidence/sample_cash_items.csv` | Internal dev-testing fixture, not client-facing |
| `apps/cash-confidence/sample_recurring_expenses.csv` | Internal dev-testing fixture, not client-facing |
| `apps/cash-confidence/requirements.txt` | Pinned dependency versions |

## How It Works

1. User enters a starting cash balance and uploads two required sheets plus one optional sheet:
   - **Cash items**: one row per inflow/outflow, with `week` (1-13), `type` (inflow/outflow), `amount`, and optional `category`/`description`.
   - **Recurring expenses**: one row per expense, with `expense_name`, `weekly_amount`, and five 1-5 ratings: `core_customer_fit`, `revenue_risk_if_cut`, `roi_clarity`, `no_cheaper_alternative`, `would_approve_today`.
   - **Receivables aging** (optional): one row per customer who owes money, with `customer_name`, `amount_outstanding`, `terms_days` (what the stated terms are, net-30 is 30), `days_outstanding` (how long it's actually been unpaid).
2. `load_cash_items()`, `load_expenses()`, and `load_receivables()` normalize and validate the sheets.
3. `_weekly_balances()` walks the 13 weeks, netting inflows and outflows against the running balance, to build the baseline forecast.
4. `run_decision_filter()` averages the five ratings per expense; anything below a 3.0 average is a cut candidate, with the specific weak questions named.
5. `run_receivables_check()` computes `days_overdue = days_outstanding - terms_days` (clipped at 0) per customer, and surfaces every account that's actually overdue against its own terms, sorted by dollar amount.
6. `run_cash_confidence()` builds an adjusted forecast: any cash-item outflow row whose `category` or `description` matches a cut-candidate expense name is dropped from week 2 onward (week 1 is left alone since that spend is already committed), then the forecast is rebuilt.
7. The app charts both forecasts, flags any week where the running balance goes negative, surfaces the receivables-timing total if any account is overdue, and lists every expense with its filter score and reasoning.

## Configuration

No API keys needed. Runs entirely locally on Python + Streamlit + pandas + openpyxl.

## Common Operations

**Start the app:**
```bash
cd apps/cash-confidence
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

**Test the forecast and filter logic directly:**
```bash
source .venv/bin/activate
python3 -c "
import pandas as pd
from analysis import run_cash_confidence
cash = pd.read_csv('sample_cash_items.csv')
expenses = pd.read_csv('sample_recurring_expenses.csv')
print(run_cash_confidence(cash, expenses, starting_balance=2000))
"
```

## Dependencies

- **Depends on:** Python 3.9+, streamlit, pandas, openpyxl.
- **Used by:** nothing yet. Planned: embed into the program platform once it's built, used in week 6 of delivery.

## Open items

- The cut-candidate match between cash items and recurring expenses relies on the owner using the same name in both sheets (category or description). A real cohort may need a cleaner linking mechanism than name matching.
- The 3.0 filter pass threshold is a starting default, not yet validated against a real cohort.
- Not yet deployed anywhere public. Currently runs locally only.
- No shared `.venv` checked in yet; run the setup steps above before first use.
